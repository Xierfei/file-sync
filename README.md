# 文件同步工具

基于 HTTP 的文件同步工具，分为服务端和客户端，均带有 Web 配置界面。适用于车载等网络不稳定场景。

## 功能特性

- **多目录管理**：服务端可配置多个目标文件夹
- **MD5 校验**：上传前比对文件 MD5，相同文件自动跳过
- **冲突处理**：重名但内容不同的文件自动加时间戳重命名
- **分段传输**：文件按 10MB 分片上传
- **断点续传**：中断后自动恢复未完成的分片
- **网络重试**：指数退避重试（最多 5 次，最长等待 60 秒）
- **实时监控**：显示当前网速、同步进度、待同步/已完成/失败列表
- **目录浏览器**：可视化选择本地和服务端目录

## 快速开始

### 安装依赖

```bash
pip install -r requirements.txt
```

### 启动服务端

```bash
cd server
python server.py
```

浏览器打开 http://localhost:8080

### 启动客户端

```bash
cd client
python client.py
```

浏览器打开 http://localhost:8081

## 使用流程

1. **服务端**：打开 Web 界面，添加目标文件夹，点击"扫描 MD5"生成缓存
2. **客户端**：填入服务端地址（如 `http://192.168.1.100:8080`），选择本地目录和服务器目标文件夹，点击"开始同步"

## 项目结构

```
file-sync/
├── requirements.txt
├── server/
│   ├── server.py              # 服务端主程序（端口 8080）
│   └── templates/
│       └── index.html         # 服务端 Web 界面
└── client/
    ├── client.py              # 客户端主程序（端口 8081）
    └── templates/
        └── index.html         # 客户端 Web 界面
```

## 技术说明

- **MD5 缓存**：服务端在目标文件夹下生成 `.md5/` 目录，结构与源文件目录一致，每个文件记录对应的 MD5 值
- **分片上传**：文件直接写入目标位置（预分配空间 + seek 写入），完成后原子 rename，无需合并步骤
- **断点续传**：服务端持久化上传会话到 `upload_sessions.json`，客户端初始化时获取已接收分片列表并跳过
